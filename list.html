Сформировать список запчастей

Контекст задачиВ документе “заказ-наряд” должны быть зафиксированы запчасти, которые необходимы для ремонта автомобиля.
Заинтересованные стороныМастер СТО, менеджер СТО, работник склада
Ключевые источники информацииТребования к системе
Атрибуты качества и ограничения (НФТ)

Критерии приемки результата

  1. При введении нескольких символов из названия запчасти в строку поиска система выдает список возможных запчастей для выбранного авто , содержащих данное сочетание символов.
  1. При нажатии кнопки “Добавить” выбранная запчасть попадает в список работ.
  1. Есть возможность выбирать и добавлять запчасти из запчастей неограниченное количество раз.
  1. Существует кнопка “Запросить необходимую запчасть”
  1. При нажатии кнопки “Запросить необходимую запчасть” открывается поля ввода названия запчасти.
  1. После нажатия кнопки “Запросить запчасть” на экране ввода система инициирует заявку на добавление запчасти в каталог.

Описание решения

Приложение 1. UC “Сформировать список запчастей”

Основной сценарий:

  1. Мастер начал вводить название запчасти в строку поиска.

    1.1 Запчасти не оказалось в списке.

  1. Система выдала список возможных запчастей для данного авто, содержащих данное сочетание символов.
  1. Мастер выбрал из предложенного списка необходимую запчасть.
  1. Мастер добавил выбранную запчасть в список необходимых запчастей.
  1. Мастер ввел количество необходимых запчастей данного вида.
  1. Мастер повторил шаги 1-5 необходимое количество раз.
  1. Мастер сохранил необходимые запчасти в заказ-наряде.

Альтернативные сценарии

1.1 Запчасти не оказалось в списке

1.1.1 Мастер запрашивает необходимую запчасть.

1.1.2 Мастер вводит название необходимой запчасти.

1.1.3. Мастер подтверждает запрос запчасти.

1.1.4 Мастер сохраняет запрошенную запчасть.

1.1.5 Система отражает введенную в п.1.1.2 запчасть серым цветом.

1.1.6 Система создает заявку на добавление запчастей, добавленных мастером в пунктах 1.1.1 - 1.1.5, в каталог.
